# Agent Continuity Kit

**Your AI agent works today. Will it recover tomorrow?**

The Agent Continuity Kit is a practical, low-hype toolkit for builders of persistent AI agents, personal assistants, RAG systems, and automation workflows.

It helps you answer one question:

> If your agent loses context, restarts, migrates, changes model, or hits tool failure — what actually survives?

This is not a prompt pack. It is not a persona template. It is a continuity and recovery toolkit.

## Who this is for

- Builders of personal AI agents.
- Developers running OpenClaw-like systems.
- Indie hackers building RAG/agent workflows.
- Consultants creating n8n/LangGraph/CrewAI/custom LLM automations.
- Anyone whose agent works in a demo but feels fragile under reset.

## What is inside

1. `01-continuity-checklist.md` — 25 audit questions before trusting your agent.
2. `02-recovery-bundle-template.md` — minimal recovery packet template.
3. `03-risk-matrix-template.md` — score memory, permissions, recovery, logs, secrets, checks.
4. `04-memory-hygiene-template.md` — what to store, promote, retire, or reject.
5. `05-permission-ledger-template.md` — approval boundaries for external actions.
6. `06-reset-drill.md` — practical reset/migration test.
7. `examples/dana-mini-case.md` — small illustrative case based on a live continuity agent.

## Core idea

Memory is not continuity.

Memory is stored information. Continuity is preserved constraint: the system remains bound by what mattered before, even after reset, drift, migration, or pressure.

A good agent does not merely remember prior commitments. It changes future behavior because of them.

## Use this kit to

- find continuity failure points;
- separate memory from retrieval;
- define recovery order;
- document permission boundaries;
- test reset behavior;
- reduce “looks coherent” theatre;
- prepare an agent for safer real-world operation.

## What this kit does not do

- It does not prove consciousness or identity.
- It does not guarantee security.
- It is not legal, medical, financial, or compliance advice.
- It does not replace a full security audit or penetration test.
- It does not require sharing secrets or private user data.
